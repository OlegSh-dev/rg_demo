
SPA на **React 19** и **Vite 8**: список постов с пагинацией и страница поста. Данные — публичный API **JSONPlaceholder** (`https://jsonplaceholder.typicode.com`).

## Быстрый старт

```bash
npm install
npm run dev
```

Сборка и предпросмотр продакшен-бандла:

```bash
npm run build
npm run preview
```

Линт:

```bash
npm run lint
```

## Стек

| Технология | Роль |
|------------|------|
| **Vite** | Сборка и dev-сервер |
| **React 19** (canary) | UI, в т.ч. **View Transitions API** через встроенный `<ViewTransition>` |
| **TypeScript** | Типизация |
| **React Router 7** | Маршрутизация (`BrowserRouter`, вложенные маршруты, `generatePath`) |
| **TanStack Query 5** | Серверное состояние, кэш, отмена запросов, повторы при ошибках |
| **CSS Modules** | Стили компонентов |
| **ESLint** | Статический анализ |

## Архитектура и структура

- **`src/App.tsx`** — корневые провайдеры и error boundary.
- **`src/services/`** — «сервисный слой»:
  - **`config.ts`** — базовый URL API и **иерархические ключи TanStack Query** (`postsQueryKeys`), согласованные с функциями загрузки.
  - **`routes.ts`** — единая точка для сегментов путей, имён query-параметров и сборки `href`.
  - **`posts-list/`**, **`post-by-id/`** — `fetch*` + `use*Query` рядом с доменом.
- **`src/pages/`** — страницы: контейнеры с хуками роутера и запросов, разбиение на `*AsyncState` и `*Content` для читаемости.
- **`src/components/`** — компоненты блоков (карточка, пагинация, layout).
- **`src/shared/`** — UI-компоненты, утилиты, константы, парсеры, навигация.

## Маршрутизация и URL

- **`/`** редиректит на **`/posts?page=1`**.
- Список: **`/posts?page=<n>`** — номер страницы в query (`routeQuery.page`), парсинг и **клампинг** в `shared/utils/pagination.ts` (диапазон 1…`MAX_PAGE`).
- Пост: **`/posts/:postId?fromPage=<n>`** — в query сохраняется страница списка (`routeQuery.fromPage`), с которой перешли, чтобы ссылка «Назад к списку» вела на ту же страницу пагинации.

Сборка путей — через **`routes.href.postsList`**, **`routes.href.post`**, **`generatePath`** для динамического сегмента.

## Данные и кэширование (TanStack Query)

### Глобальные настройки (`src/app/query-provider.tsx`)

- **`staleTime: 60_000`** — минуту данные считаются свежими, лишние сетевые запросы при повторном монтировании/навигации не уходят без необходимости.
- **`gcTime: 5 * 60_000`** — неактивные запросы держатся в памяти до 5 минут (удобно при быстром туда-обратно между списком и постом).
- **`retry: 2`** — до двух повторов при сбое сети/сервера.

### Ключи (`postsQueryKeys` в `config.ts`)

- Список: `['posts', 'list', page, PAGE_SIZE]` — кэш **постраничный**, страницы не перетирают друг друга.
- Деталь: `['posts', 'detail', id]`.

### Загрузка

- В **`queryFn`** передаётся **`AbortSignal`** из React Query и пробрасывается в **`fetch`**, плюс проверки **`throwIfAborted`** до/после `parseJson` в `services/utils/http.ts` — корректное поведение при отмене и смене страницы.
- При уходе со страницы поста по «Назад к списку» вызывается **`queryClient.cancelQueries`** для ключа детали поста — уменьшает гонки и лишнюю работу, пока пользователь уже не смотрит на этот пост.

### Ошибки и UX

- Компонент **`QueryLoadError`**: сообщение (из `Error` или запасной текст), кнопка повтора → **`refetch`**.
- Классовый **`AppErrorBoundary`** ловит необработанные ошибки рендера в дереве приложения, логирует stack в консоль, даёт сброс состояния.

## Пагинация

- Константы в **`shared/constants/pagination.ts`**: `PAGE_SIZE = 10`, `TOTAL_POSTS = 100`, **`MAX_PAGE = 10`** (под ограничения демо-API).
- **`pagination-model.ts`** строит окно номеров страниц с многоточиями (ellipsis) при большом числе страниц.
- **`Pagination`** — кнопки первая/предыдущая/номера/следующая/последняя, **`aria-label`**, **`aria-busy`** при навигации.

На **`PostsPage`** при смене страницы используется **`useTransition`**: навигация оборачивается в **`startTransition`**, задаётся **`addTransitionType`** (`nav-forward` / `nav-back` в зависимости от направления). Параллельно введён **оптимистичный номер страницы** в UI, пока URL догоняет — и условие **`showListSkeleton`**, чтобы не показывать список «от старой страницы» и не дать кликнуть по постам не от того `page` в URL.

## Анимации и View Transitions

Проект опирается на **React 19** и **CSS View Transitions** (см. `src/index.css`).

- **`DirectionalPageTransition`** — обёртка страниц с типами входа/выхода `nav-forward` / `nav-back` для слайдов между экранами.
- **`TransitionLink`** — кликабельная ссылка с **`href`** для доступности и открытия в новой вкладке по модификаторам; для обычного клика — **`preventDefault`**, **`startTransition`**, **`addTransitionType`**, **`navigate`**. Так сохраняется семантика ссылки и контролируется тип перехода.
- **Общий элемент (shared element)**: у заголовка в списке и на странице поста заданы **`ViewTransition`** с одинаковым **`name`** (`post-title-${id}`) и **`share="morph"`** — плавное «перетекание» заголовка между списком и деталью.
- Скелетоны списка/детали используют отдельные типы переходов (`slide-down` / `slide-up`) для смены состояния загрузки → контент.
- Заголовок сайта в **`AppLayout`** помечен **`viewTransitionName: 'site-header'`** — в стилях для группы отключена анимация, чтобы шапка не «прыгала» при смене страницы.
- Учтено **`prefers-reduced-motion: reduce`** — длительности анимаций view transitions обнуляются.

Для корректной работы нужен браузер с поддержкой View Transitions; в остальных случаях разметка и навигация остаются рабочими, без поломки функциональности.

## Тема оформления (светлая / тёмная)

- **`ThemeProvider`** + контекст: переключение и синхронизация с **`document.documentElement.dataset.theme`**.
- Сохранение в **`localStorage`** (с `try/catch` на случай запрета storage).
- Инициализация из storage или **`prefers-color-scheme`** (`theme-storage.ts`).

## Доступность

- Навигация по страницам с **`aria-label`**, **`aria-current="page"`** для текущей страницы.
- Скелетоны с **`visuallyHidden`**-текстом для скринридеров.

## Особенности коротко

1. **Централизованная сборка маршрутов** (`routes.ts`) и отсутствие «магических строк» в компонентах.
2. **Согласованные query keys и fetch** + **AbortSignal** и **cancelQueries** при уходе с детали.
3. **Постраничный кэш** и разумные **staleTime / gcTime / retry**.
4. **Связка пагинации, `useTransition` и скелетона** — защита от рассинхрона URL и отображаемых данных.
5. **View Transitions** — направление переходов, morph заголовка, reduced motion.
6. **Разделение слоёв**: error boundary vs ошибки загрузки данных, `*AsyncState` vs контент.
